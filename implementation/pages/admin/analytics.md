# Admin Analytics & BI Implementation Plan

**Component**: `AdminAnalytics.tsx`
**Route**: `/admin/analytics`

## Purpose
Business Intelligence dashboard for tracking SaaS health and growth metrics.

## Features
*   [x] **MRR/ARR**: Track recurring revenue trends.
*   [x] **User Growth**: Signups vs Churn over time.
*   [x] **Traffic Usage**: Total system-wide credit consumption.

## Planned Improvements
*   [ ] **Retention Cohorts**: Visualization of user retention by signup month.
*   [ ] **LTV Calculator**: Average Lifetime Value per user segment.
*   [ ] **Churn Analysis**: Breakdown of churn reasons and voluntary vs involuntary.
*   [ ] **Feature Usage**: Heatmap of most used dashboard pages.
*   [ ] **Conversion Funnel**: Step-by-step drop-off from Signup -> First Deposit.
*   [ ] **Revenue Forecast**: AI-based projection of next month's revenue.
*   [ ] **Cost Analysis**: Server/Proxy costs vs Revenue (Gross Margin).
*   [x] **Active Users (DAU/MAU)**: Daily and Monthly Active User counts.
*   [ ] **Support Load**: Correlation between new features and ticket volume.
*   [ ] **Geographic Heatmap**: User density by country.
*   [ ] **Device Stats**: Customer browser/OS breakdown.
*   [ ] **Referral Performance**: Top referring domains/affiliates.
*   [ ] **Credit Burn Rate**: How fast users spend their deposits.
*   [ ] **Plan Distribution**: Pie chart of users by subscription tier.
*   [ ] **Failed Payments**: Rate of declined transactions.
*   [ ] **Recovery Rate**: Success of "dunning" emails (payment retries).
*   [ ] **Export Reports**: PDF/CSV download of monthly executive summary.
