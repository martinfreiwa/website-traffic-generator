# Admin Support & Broadcasts Implementation Plan

**Components**: `AdminTickets.tsx`, `AdminBroadcasts.tsx`
**Routes**: `/admin/tickets`, `/admin/broadcasts`

## Purpose
Communication with users.

## Features
*   [ ] **Support Tickets**: Respond to user queries.
*   [ ] **Broadcasts**: detailed system-wide announcements (e.g., maintenance).

## Planned Improvements
*   [ ] **Canned Responses**: Library of quick reply templates.
*   [ ] **Ticket Assignment**: Assign tickets to specific admin team members.
*   [ ] **Internal Comments**: Private comments on tickets.
*   [ ] **Bulk Close**: Select and close multiple old tickets.
*   [ ] **Macros**: Quick key shortcuts for common replies.
*   [ ] **Satisfaction Survey**: Auto-email "Rate this conversation" link on close.
*   [ ] **Ticket Merge**: Combine duplicate tickets from the same user.
*   [ ] **SLA Timer**: "Time until breach" countdown for VIP tickets.
*   [ ] **Sentiment Analysis**: AI indicator "User is Angry" on ticket list.
*   [ ] **User History**: "User has opened 5 tickets in last 24h" warning.
*   [ ] **Rich Text**: Bold/Underline support in replies.
*   [ ] **Collision Detection**: "Admin X is also viewing this ticket".
*   [ ] **Ticket Snooze**: "Remind me in 24 hours" button.
*   [ ] **Auto-Assign**: Round-robin assignment to available agents.
*   [ ] **Tagging System**: Add internal tags like "Bug", "Refund", "VIP".
*   [ ] **Forward Ticket**: Email ticket content to external address (e.g., Dev team).
*   [ ] **Spam Filter**: "Mark as Spam" button for junk.
*   [ ] **Audit Log**: "Admin Y changed priority to High".
*   [ ] **Broadcast Scheduling**: Set date/time for broadcast to go live.
*   [ ] **Broadcast Targeting**: "Send only to users with >$100 spend".
*   [ ] **Template Manager**: CRUD interface for Canned Responses.
*   [ ] **Ticket Analytics**: Bar chart of tickets opened per day.
*   [ ] **Resolution Time**: Metric for "Avg time to close".
*   [ ] **Agent Leaderboard**: "Most tickets closed" ranking.
*   [ ] **Keyword Alert**: Highlight "refund", "sue", "urgent" in red.
*   [ ] **User Profile Card**: Mini user profile side-panel in ticket view.
*   [ ] **Previous Tickets**: List of past tickets from this user in sidebar.
*   [ ] **One-Click Refund**: Button to trigger refund directly from ticket.
*   [ ] **Draft Verify**: Require senior admin approval for draft replies.
*   [ ] **Emoji React**: Add emoji reaction to user message (internal only).
*   [ ] **Language Detect**: "User is speaking German" indicator.
*   [ ] **Auto-Translation**: "Translate message to English" button.
*   [ ] **Ticket Viewers**: "X users watching" indicator.
*   [ ] **Private Note Ping**: @mention another admin in internal notes.
*   [ ] **Similar Tickets**: "3 other users reported this today" (incident detection).
*   [ ] **Broadcast Stats**: "500 reads, 20 clicks" on announcement.
*   [ ] **Emergency Alert**: "Send SMS to all admins" button.
*   [ ] **User Local Time**: "It is 3am for this user" warning.
*   [ ] **Device Info**: Parse User-Agent string to show "Mac / Chrome".
*   [ ] **IP Ban**: Button to ban user IP directly from ticket.
*   [ ] **Ticket Source**: Show if via Email, Web, or API.
*   [ ] **File Preview**: Inline PDF/Image viewer for attachments.
*   [ ] **Bulk Tagging**: Select 50 tickets and add "Feature Request" tag.
*   [ ] **Knowledge Base Link**: "Insert link to Article X" search bar.
*   [ ] **Custom Fields**: "Order ID" field populated from form.
*   [ ] **Priority Color**: High=Red, Medium=Yellow, Low=Green indicators.
*   [ ] **Filter Views**: "My Tickets", "Unassigned", "SLA Breached".
*   [ ] **Export Thread**: Download single conversation as PDF.
*   [ ] **Webhook Trigger**: Fire webhook on new ticket creation.
